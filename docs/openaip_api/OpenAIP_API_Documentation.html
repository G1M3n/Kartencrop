OAS 3.0

https://api.core.openaip.net/api/system/specs/v1/schema.json

The Core API provides endpoints to handle and consume a variety of aeronautical content.

Authentication

To utilize the OpenAIP public API, you'll need a unique API key. Here's how to obtain and use your client API key:

  1. Ensure you are logged into your OpenAIP account
  2. Navigate to the user profile's API Clients page
  3. On the API Clients page, request a new API client. Each client comes with its own unique API key
  4. When making requests to the OpenAIP public API, include your API key using one of the following methods:

    • HTTP Header: Use the HTTP header x-openaip-api-key to include the API key in each request to the API.
    • Query Parameter: Use the query parameter apiKey to include the API key in each request to the API.

Keep your API keys confidential at all times! Treat them like passwords as they grant access to OpenAIP resources on your behalf. The API Clients page allows you to view or revoke your API keys at any time. If you encounter any issues or have questions, please contact our support team for assistance.

If you want to provide more sophisticated authentication mechanisms with OpenAIP inside your application, for example OIDC/OAuth2, please contact us directly for more information.

Rate Limiting

The OpenAIP API enforces rate limits on several endpoints. Please consider caching responses on your side to not hit the rate limit.

Attribution

Please add a proper attribution link to OpenAIP (https://www.openaip.net) as data source within your application!
